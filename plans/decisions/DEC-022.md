# DEC-022: Fix Hardcoded Direction in Signal Classification

**Date:** May 13, 2026
**Status:** ACTIVE
**Severity:** CRITICAL — breaks all BEARISH signals

## Problem

`signalClassification.service.ts` line 112 has `const direction: 'bullish' | 'bearish' = 'bullish';` hardcoded. This means:
- All signals are treated as bullish regardless of actual verdict
- BEARISH signals get TP from resistance above (wrong) and SL from swing low (wrong)
- RR calculation is semantically incorrect for all sell signals

## Decision

Derive direction from the signal's actual verdict passed via params:
```typescript
const direction = params.direction ?? deriveDirectionFromVerdict(params.verdict);
```
Where `deriveDirectionFromVerdict` maps:
- STRONG_BULLISH, BULLISH → 'bullish'
- STRONG_BEARISH, BEARISH → 'bearish'
- SIDEWAYS → 'bullish' (default, no signal expected anyway)

## Impact

Files modified: `signalClassification.service.ts`
Calling code in `aiWorkflow.cron.ts` must pass direction or verdict to `classifySignal()`.
Zero breaking changes to exports — only internal logic fix.

## Guardrails

- Function signature may add optional `direction` param (backward compatible)
- If direction cannot be derived, default to 'bullish' (safe fallback, same as current broken behavior)
- All existing test cases for bullish signals remain valid
