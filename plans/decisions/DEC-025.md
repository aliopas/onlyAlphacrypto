# DEC-025: Raise VOLATILE Regime ATR Threshold from 3% to 5%

**Date:** May 13, 2026
**Status:** ACTIVE
**Severity:** MEDIUM — over-triggers in normal crypto volatility

## Problem

`marketRegime.service.ts` line 109 triggers `volatilityScore = 1` when `atrPercent > 3%`. For BTC at $100k, ATR > $3000 = 3% is common during normal market conditions. This causes VOLATILE regime which blocks ALL signals too frequently.

In crypto, 3% daily ATR is normal. 5%+ is genuinely volatile.

## Decision

Raise thresholds:
- `volatilityScore = 1` (VOLATILE trigger): ATR > 5% (was 3%)
- `volatilityScore = 0.5`: ATR > 2.5% (was 1.5%)
- `volatilityScore = 0`: ATR <= 2.5%

Update VOLATILE regime check (line 146): `atrPercent > 6 || volatilityScore >= 1` (was 4 and 1)

## Impact

Files modified: `marketRegime.service.ts` lines 109, 146
Zero breaking changes — only threshold constants change.
SIDEWAYS regime threshold remains unchanged.

## Guardrails

- VOLATILE regime still exists and still blocks ALL signals when triggered
- Only the trigger threshold changes — the response is the same
- Fear & Greed extreme values (< 25) still trigger RISK_OFF independently
- Macro keyword triggers (war, sanctions, fed) still work unchanged
