# DEC-028: Fix Algorithm Verdict in Shadow Mode — Composite Direction Logic

**Date:** May 14, 2026
**Status:** APPROVED
**Severity:** CRITICAL — Shadow Mode produces 100% NEUTRAL, making the entire comparison meaningless
**Decision Authority:** Tech Lead

---

## Problem Statement

Shadow mode dashboard shows ALL 7 algorithm outputs as `NEUTRAL` while AI produces `BUY`/`SELL`. This means:

1. Algorithm agreement = always `false`
2. Algorithm PnL = always `0` (calculatePnl returns 0 for NEUTRAL)
3. Algorithm can never "win" a shadow comparison
4. Shadow mode data is completely useless for decision-making

## Root Cause

**File:** `backend/src/crons/aiWorkflow.cron.ts`
**Line 773:** `const algoVerdict = mapTrendToVerdict(taResult.trend)`

The algorithm verdict is derived **exclusively** from `taResult.trend` which is the output of `detectTrend()` — a function that checks 4H EMA alignment only. `detectTrend()` has 9 return paths, **8 of which return SIDEWAYS**. In crypto markets, 4H EMAs are rarely perfectly aligned, so the result is almost always SIDEWAYS → NEUTRAL.

Meanwhile, the algorithm already produces rich analysis in `taResult`:
- `structure.pattern` — BOS_BULLISH, BOS_BEARISH, CHOCH_BULLISH, CHOCH_BEARISH, FAILED_BOS, HH_HL, LH_LL
- `structure.trend` — 'bullish' | 'bearish' | 'sideways'
- `candlePattern.direction` — 'bullish' | 'bearish' | null
- `volume.isAboveAverage` — volume confirmation
- `qualityScore.score` — 0-100 composite quality

**NONE of these are used for the shadow mode algorithm verdict.**

## Decision

Replace the single-factor EMA trend verdict with a **composite algorithm verdict** that uses the full intelligence already computed by `analyzeTechnicals()`.

## Composite Verdict Logic (Priority Chain)

```
1. STRUCTURE GATE (highest confidence)
   - BOS_BULLISH / HH_HL → BULLISH
   - BOS_BEARISH / LH_LL → BEARISH
   - CHOCH_BULLISH → BULLISH (with lower confidence)
   - CHOCH_BEARISH → BEARISH (with lower confidence)
   - FAILED_BOS → NEUTRAL (hard reject per DEC-017)
   - NONE → fall through to #2

2. CANDLE PATTERN (if structure = NONE)
   - direction = 'bullish' AND isValid = true → BULLISH
   - direction = 'bearish' AND isValid = true → BEARISH
   - else → fall through to #3

3. EMA TREND (lowest confidence — existing logic)
   - STRONG_BULLISH / BULLISH → BULLISH
   - STRONG_BEARISH / BEARISH → BEARISH
   - SIDEWAYS → NEUTRAL

4. QUALITY GATE (overrides)
   - qualityScore < 40 → NEUTRAL (weak analysis, don't trust direction)
   - qualityScore >= 40 → keep verdict from steps 1-3
```
