# DEC-029: Fix Quality Threshold Discrepancy in Shadow Mode Algorithm Verdict

**Date:** May 14, 2026
**Status:** APPROVED
**Severity:** CRITICAL — Shadow Mode produces 100% NEUTRAL due to threshold mismatch
**Decision Authority:** Tech Lead
**Supersedes:** Partial implementation of DEC-028

---

## Problem Statement

DEC-028 approved `qualityScore < 40 → NEUTRAL` for shadow mode algorithm verdict. Implementation uses `< 60` instead. This single-line discrepancy causes ALL algorithm verdicts to be forced to NEUTRAL, making shadow mode comparison data meaningless.

## Root Cause

**File:** `backend/src/crons/aiWorkflow.cron.ts`
**Line 126:** `if (taResult.qualityScore.score < 60) return 'NEUTRAL'`

DEC-028 specification says `< 40`. Code says `< 60`. Typical quality scores range 0-50, all below 60.

## Decision

Change quality threshold from `< 60` to `< 40` in `deriveAlgorithmVerdict()` line 126.

## Rationale

1. DEC-009 (`quality < 60 = rejected`) governs **live signal pipeline** (radar_signals), NOT shadow mode
2. Shadow mode purpose = comparison/testing — needs directional data even at low confidence
3. Quality 40-59 = "weak but directional" — valid for shadow comparison tracking
4. Quality < 40 = truly unreliable analysis → NEUTRAL is appropriate veto
5. DEC-028 was already approved with this threshold — implementation must match decision
