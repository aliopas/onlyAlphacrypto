# DEC-024: Fix Daily Trend Gate to Allow Directional Alignment

**Date:** May 13, 2026
**Status:** ACTIVE
**Severity:** HIGH — logical error blocking valid BEARISH signals + aborting all processing

## Problem

Two bugs in `aiWorkflow.cron.ts` lines 659-669:

**Bug A:** Daily trend check blocks BEARISH/STRONG_BEARISH coins unconditionally. A SELL signal on a bearish-trend coin is the CORRECT trade (trend-aligned), but the gate blocks it. The check should only block signals that go AGAINST the trend.

**Bug B:** The `continue` statement on line 665 aborts ALL remaining processing for the item — including article saving, memory updates, news history, and cache invalidation. It should only skip signal creation, not the entire item.

## Decision

**Fix A:** Replace blanket bearish rejection with directional alignment check:
- BULLISH/STRONG_BULLISH signal + BEARISH/STRONG_BEARISH trend → SKIP (counter-trend)
- BEARISH/STRONG_BEARISH signal + BEARISH/STRONG_BEARISH trend → ALLOW (trend-aligned)
- SIDEWAYS trend → ALLOW (no directional bias)

**Fix B:** Replace `continue` with a `shouldSkipSignal` flag. Signal creation is skipped, but article/memory/news/cache processing continues.

## Impact

Files modified: `aiWorkflow.cron.ts` lines 659-669 + surrounding signal creation block
Requires access to signal direction (from verdict) before the gate check.
Zero breaking changes to other services.

## Guardrails

- STRONG_BEARISH trend still blocks BULLISH signals (counter-trend protection maintained)
- SIDEWAYS trend allows all directions (no bias)
- Article processing, memory saves, and news history are NEVER skipped
- Daily trend detection logic in `dailyTrend.service.ts` is unchanged
