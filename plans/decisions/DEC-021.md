# DEC-021: Fix TP/SL V2 RR Fallback Math

**Date:** May 13, 2026
**Status:** ACTIVE
**Severity:** CRITICAL — guarantees 100% rejection when no S/R levels exist

## Problem

`tpslCalculatorV2.service.ts` ATR fallback produces RR=1.5 (TP=1.5×ATR / SL=1.0×ATR). Percentage fallback produces RR=1.875 (TP=15% / SL=8%). Both are below tactical minimum RR of 2.0. When no S/R levels with strengthScore >= 60 exist (common), ALL signals are mathematically guaranteed to be rejected.

## Decision

Change ATR multipliers to produce RR >= 2.0 by default:
- TACTICAL: TP = 2.0× ATR, SL = 1.0× ATR → RR = 2.0
- STRATEGIC: TP = 3.0× ATR, SL = 1.0× ATR → RR = 3.0

Change percentage fallbacks:
- TACTICAL: TP = 12%, SL = 5% → RR = 2.4
- STRATEGIC: TP = 18%, SL = 5% → RR = 3.6

## Impact

Files modified: `tpslCalculatorV2.service.ts` lines 79-80, 113-118
Zero breaking changes — only internal multiplier values change.
RR gate in `tpslSanityGate.service.ts` and `signalClassification.service.ts` remain untouched.

## Guardrails

- S/R-based TP/SL (priority 1) remains unchanged — algorithm still prefers real levels
- Only fallback paths (ATR and percentage) change
- TP/SL distance bounds (1%-40%) in sanity gate still enforced
